# Display Name Hash + Per-Installation Salt

Module: [Persistence](../08-persistence.md)

Description:
Generate a per-installation salt via WebCrypto `getRandomValues`
and persist it under IndexedDB `hr-profile.keys`. Compute
`playerHash = xxh64(displayName + localSalt)` and write
`metadata.playerHash` + `metadata.playerLabel` into save records.
The salt is regenerated by `WIPE_LOCAL_DATA scope=profile|all`.
Per [`data-inventory.md`](../../../docs/architecture/data-inventory.md)
and [`persistence.md`](../../../docs/architecture/persistence.md).

Acceptance Criteria:
- `getOrCreateLocalSalt()` generates a 16-byte salt on first call;
  subsequent calls return the persisted bytes.
- `computePlayerHash(displayName)` returns a 16-hex-char digest.
- `WIPE_LOCAL_DATA scope=profile|all` rotates the salt; the next
  hash for the same display name differs.
- Default `displayNameMode` is `hashed`; existing saves migrate by
  recomputing the hash from the loaded `playerName` on first load
  and rewriting on next save.
