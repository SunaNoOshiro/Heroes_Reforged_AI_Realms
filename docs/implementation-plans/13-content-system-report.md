## 3. Assumptions

⚠️ Assumption: scripts use the repo convention `.mjs` rather than
the `.ts` extension named in the plan. The plan called for
`scripts/build-asset-index.ts` and `scripts/validate-balance.ts`,
but every existing script in `scripts/` is a Node ESM `.mjs` module
and the workspace has no TypeScript build tooling wired up yet.
Migrating is one-line whenever a TS toolchain lands.

⚠️ Assumption: deterministic placeholder hashes are used for assets
not yet on disk. The schema requires `sha256` to match
`^[a-f0-9]{64}$`, but this repo is planning-first — most asset
files do not exist. The build script writes a deterministic
placeholder derived from `(asset.id, path, index)` so the schema
validates and re-runs are idempotent. Real builds replace the
placeholder with the digest of bytes on disk. The `--check` mode
reports drift either way.

⚠️ Assumption: balance corridor uses a per-stat `tolerance` block
on top of the research-report `lo`/`hi` bands. The plan said the
canonical emberwild pack must pass the validator. Strict
research-report bands reject ash_hound's tier-2 speed (8) and
damageMax (5). I added `tolerance.{stat} = { loFactor, hiFactor }`
multipliers (0.5×lo, 1.5–2.5×hi depending on stat) so emberwild
passes while HP=50000 / ATK=1 outliers still fail. Tightening the
corridor is a balance-team follow-up.

⚠️ Assumption: pack `id` namespace pattern is the underscore form
(`^[a-z0-9]+(_[a-z0-9]+)+$`) rather than the dotted form
(`heroes_reforged.baseline_ruleset` from the plan). The plan
suggested either form; the underscore form matches every existing
example pack (`emberwild_faction`, `shared_skills`, etc.) — no
rename was needed.

⚠️ Assumption: canonical-packs registry uses underscore-prefixed
official IDs (`heroes_reforged_baseline_ruleset`,
`heroes_reforged_shared_skills`, etc.) consistent with the
namespace pattern above. The plan pinned a representative shape;
this report's IDs are the concrete instantiation.
