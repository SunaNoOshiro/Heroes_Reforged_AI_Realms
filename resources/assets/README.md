# resources/assets

Concrete asset files can live here once the project starts importing or
authoring visuals and audio.

Examples:

- unit sprite atlases
- hero portraits and paper-doll layers
- town screen overlays
- spell VFX sheets
- building and map-object animations
- sound effects, voice, and ambient loops

Schema records should still reference stable asset IDs. The actual file
paths belong in manifests or asset indexes, not in gameplay records.
